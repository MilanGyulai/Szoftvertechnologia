Ebben a dokumentumban megtalálható lesz a játék Skeletonja, tehát az alapvető kódok a programhoz.

Inventory osztály

            public class Inventory {

                private List<Item> items;
                private int capacity;     
            

                public Inventory(int capacity) {

                    this.capacity = capacity;

                    this.items = new ArrayList<>();

                }

            

                
                public boolean addItem(Item item) {

                    if (items.size() < capacity) {

                        items.add(item);

                        return true;

                    }

                    return false; // Sikertelen, nincs hely

                }

            

                
                public boolean removeItem(Item item) {

                    return items.remove(item);

                }

            

                
                public List<Item> getItems() {

                    return items;

                }

            }

Crafting osztály

                public class Crafting {

                    private Map<String, Integer> requiredMaterials; 
                    private Item craftedItem;                      
                

                    public Crafting(Map<String, Integer> requiredMaterials, Item craftedItem) {

                        this.requiredMaterials = requiredMaterials;

                        this.craftedItem = craftedItem;

                    }

                

                    
                    public boolean canCraft(Inventory inventory) {

                        for (Map.Entry<String, Integer> material : requiredMaterials.entrySet()) {

                            long count = inventory.getItems().stream()

                                    .filter(item -> item.getName().equals(material.getKey()))

                                    .count();

                            if (count < material.getValue()) {

                                return false;

                            }

                        }

                        return true;

                    }

                

                    
                    public Item craftItem() {

                        return craftedItem;

                    }

                }

                public class SkillTree {

                    public void unlockSkill(Skill skill) {

                            unlockedSkills.add(skill);

                    public List<Skill> getUnlockedSkills() {

Aktív és Passzív Objektumok

Az aktív objektumok azok, amelyek saját vezérlési szálat futtatnak, vagy folyamatosan működnek a rendszerben. Ezek biztosítják a rendszer dinamikus működését.

  • PlayerInteraction: Kezeli a játékos és az NPC-k, illetve a környezet közötti interakciókat.
  • GameController: A játék fő vezérlő mechanizmusa, amely koordinálja az alrendszerek működését.
  • MarketController: Az áruk és valuták kereskedését kezeli.
  • EnvironmentManager: Az időjárási viszonyokat, környezeti eseményeket és helyszíneket koordinálja.
  • QuestController: Feladatok és küldetések állapotát, valamint azok logikáját kezeli.

Passzív Objektumok:

  • Inventory: A játékos és az NPC-k tárgyait tartalmazza.
  • SkillTree: A játékos képességeit és fejlődési útvonalát modellezi.
  • SaveSlot: A játék aktuális állapotának tárolása.
  • Quest: Egyéni küldetéseket reprezentál.
  • Environment: A környezeti jellemzők (pl. tereptárgyak, faanyagok, bányászható ércek) tárolására szolgál.
  • Item: A világban található tárgyakat, felszereléseket írja le.

Láthatóság:

GameController: Minden objektum számára elérhető, mint központi vezérlő.

PlayerInteraction: Csak az NPC, Inventory, SkillTree, és Quest objektumokat látja.

QuestController: Látja a Quest, PlayerInteraction, és GameController objektumokat.

MarketController: Látja a Inventory, Item, és PlayerInteraction objektumokat.

EnvironmentManager: Látja az Environment és a GameController objektumokat.

Üzenetváltások:

  • PlayerInteraction ↔ NPC:

    A játékos beszélgetést kezdeményez egy NPC-vel:

    Kérés: "Van valamilyen küldetésed?"

    Válasz: "Igen, gyűjts 5 követ!"

  • PlayerInteraction ↔ NPC:

    QuestController -> PlayerInteraction:

    A játékos befejez egy küldetést:

    Üzenet: "Küldetés teljesítve, adj át 100 XP-t a játékosnak!"

  • PlayerInteraction ↔ NPC:

    MarketController -> Inventory:

    A játékos elad egy tárgyat:

    Üzenet: "Távolítsd el az eladott tárgyat az inventory-ból."

Tesztelési terv

A játék minőségbiztosításához szükséges egy részletes, de nagyvonalú tesztelési terv, amely különböző szinteken és funkciók szerint ellenőrzi a rendszer megfelelő működését. A tesztelési terv kiterjed a rendszer alapvető funkcióira, interakcióira, és a különböző játékelemek helyes integrációjára.

Tesztelési Célok

A játék minden modulja (pl. Inventory, Quest, SkillTree) megfelelően működjön, és egymással zökkenőmentesen kommunikáljon. A felhasználói interakciók (pl. parancsok, mozgások, tárgyhasználat) pontosan és következetesen végrehajtódjanak. A rendszer stabilitása, teljesítménye és hibakezelése biztosított legyen. A játékos tapasztalatának (UX) folyamatosan magas szintű biztosítása. A játék különböző eszközökön és konfigurációk mellett is megfelelően működjön.

Tesztelési folyamatok

  • 1. Unit Tesztelés
    A játék egyes moduljainak (osztályainak) különálló tesztelése. Tesztelendő komponensek: PlayerInteraction: Ellenőrizni, hogy a játékos interakciói (beszélgetések, parancsok) megfelelő hatással vannak az NPC-kre és a világra. Inventory: Biztosítani, hogy a tárgyak hozzáadása, eltávolítása és megjelenítése helyesen működjön. QuestController: Tesztelni, hogy a küldetések állapota és teljesítési logikája megfelelően frissül. MarketController: Ellenőrizni az árucikkek vásárlását és eladását. SkillTree: Tesztelni a képességpontok elosztását és a hatásaikat. Példa Unit Tesztekre: Inventory tesztelése: Hozzáadható-e egy új tárgy? Megjeleníti-e helyesen a játékos tárgyait? Törölhető-e egy nem létező tárgy (hibakezelés)? Quest tesztelése: Aktiválódik-e egy új küldetés helyesen? Teljesítési feltételek megfelelően értékelődnek ki?
  • 2. Integrációs Tesztelés A különböző modulok együttműködésének vizsgálata. Tesztelési szempontok: Az Inventory szinkronizációja a MarketController-rel. Az Environment változásai (pl. időjárás) hatással vannak-e az NPC-kre és a játékos mozgására. A PlayerInteraction által küldött parancsok aktiválják-e a megfelelő Quest logikát.
  • 3. Funkcionális Tesztelés Az egész játék funkcionális működésének ellenőrzése. Tesztelendő funkciók: Karakter létrehozása és kezdő tulajdonságok kiosztása. Játékos mozgása és interakciói (pl. tárgyak gyűjtése, küldetések elfogadása). NPC-k működése (pl. párbeszédek, kereskedés). Harcrendszer alapvető működése (sebzés, győzelem/veszteség kimenetelek). Példa Funkcionális Tesztekre: Létrehoz-e a játék egy új karaktert alapértékekkel? Helyesen frissül-e az inventory, ha a játékos tárgyat szerez? Aktiválódik-e egy küldetés, ha a játékos beszél egy NPC-vel?
  • 4. Teljesítménytesztelés A rendszer stabilitásának és hatékonyságának tesztelése. Cél: Felmérni, hogy a játék hogyan kezeli a nagyobb terheléseket. Tesztelendő szituációk: Több mint 50 NPC mozog és interakcióba lép a környezetével. Több mint 100 tárgy jelenik meg egyszerre az inventory-ban. A játék hosszú távú futtatása (pl. 3-4 óra folyamatos játék).

Tesztelési Forgatókönyvek

Alapszintű Forgatókönyv: Játék kezdése Új karakter létrehozása. Kezdő tulajdonságpontok kiosztása. Kezdeti küldetés elfogadása egy NPC-től. Az első szörny legyőzése és jutalom megszerzése. Mentés és visszatöltés.

Speciális Forgatókönyv: Küldetéslánc tesztelése Küldetések elfogadása és teljesítése sorrendben. Küldetéslánc befejezése és jutalom megszerzése. Ellenőrzés, hogy a küldetéslánc újraindítható-e hiba nélkül.

Szélsőséges Forgatókönyv: Hibakezelés tesztelése Hibás parancsok megadása az NPC-knek. Nem létező tárgyak vásárlása a piacról. A játék kényszerített megszakítása (pl. bezárás a harc közben) és visszatöltés.

Tesztelési Eszközök és Módszerek

Automatizált Tesztek:JUnit.

Manuális Tesztek:QA tesztelők által, előre meghatározott forgatókönyvek alapján.

Log Elemzés:A játék futása közbeni hibák és események naplózása.

Stresszteszt Szkriptek:Nagy terhelés szimulálása (sok NPC, tárgy, harci esemény).
